# 前缀和

## 📚 前缀和算法介绍





### 什么是前缀和？



前缀和是一种**预处理技术**，用于快速计算数组（或列表）中**子区间（或子数组）的和**。

给定一个数组 $A$，它的前缀和数组 $P$ 的定义是：

- $P[i]$ 存储的是数组 $A$ 中从第一个元素到第 $i$ 个元素（包括第 $i$ 个元素）的所有元素的和。

用公式表示就是：



$$P[i] = A[0] + A[1] + \dots + A[i]$$



### 核心思想：空间换时间



前缀和算法的核心思想是**“空间换时间”**。

1. **预处理（O(n) 时间）：** 花费 $O(n)$ 的时间（$n$ 是数组长度）计算出整个前缀和数组 $P$。
2. **查询（O(1) 时间）：** 之后，任何子区间 $A[i \dots j]$ 的和都可以在 $O(1)$ 的时间复杂度内计算出来。

------



## 🔍 前缀和的构建与应用





### 1. 前缀和数组的构建



假设我们有一个原始数组 $A$。

- **原始数组 $A$:** $A[0], A[1], A[2], \dots, A[n-1]$
- **前缀和数组 $P$:** $P[0], P[1], P[2], \dots, P[n-1]$

**计算关系（递推公式）：**

$$P[i] = P[i-1] + A[i]$$

其中，$P[-1]$ 视为 0（通常在实现时通过索引偏移或在 $P$ 数组开头多加一个 0 来处理）。



#### 💡 具体例子（以 0-based 索引为例）



假设原始数组 $A = [3, 1, 4, 1, 5]$：

| **索引 i** | **A[i]** | **P[i] (计算过程)**    | **P[i] (值)** |
| ---------- | -------- | ---------------------- | ------------- |
| 0          | 3        | $P[-1] + A[0] = 0 + 3$ | 3             |
| 1          | 1        | $P[0] + A[1] = 3 + 1$  | 4             |
| 2          | 4        | $P[1] + A[2] = 4 + 4$  | 8             |
| 3          | 1        | $P[2] + A[3] = 8 + 1$  | 9             |
| 4          | 5        | $P[3] + A[4] = 9 + 5$  | 14            |

所以，前缀和数组 $P = [3, 4, 8, 9, 14]$。



### 2. 子数组求和的计算



我们的目标是计算子数组 $A[i \dots j]$ 的和（即从索引 $i$ 到索引 $j$ 的所有元素之和，且 $i \le j$）。

根据前缀和的定义：

- $P[j] = A[0] + \dots + A[i-1] + A[i] + \dots + A[j]$
- $P[i-1] = A[0] + \dots + A[i-1]$

因此，子数组 $A[i \dots j]$ 的和为：



$$\text{Sum}(A[i \dots j]) = P[j] - P[i-1]$$

为了简化边界处理（避免 $i=0$ 时出现 $P[-1]$），我们通常构建一个**“带 0”的前缀和数组 $S$**，长度为 $n+1$，其中 $S[0]=0$。

- $S[k]$ 存储的是数组 $A$ 中前 $k$ 个元素的和（即 $A[0]$ 到 $A[k-1]$ 的和）。
- $S[k] = A[0] + \dots + A[k-1]$



#### 💡 具体例子（使用带 0 的前缀和数组 $S$）



原始数组 $A = [3, 1, 4, 1, 5]$ (长度 $n=5$)。

| **索引 k (在 S 中)** | **A 中元素个数** | **S[k] (值)** |
| -------------------- | ---------------- | ------------- |
| 0                    | 0                | 0             |
| 1                    | 1 ($A[0]$)       | 3             |
| 2                    | 2 ($A[0..1]$)    | 4             |
| 3                    | 3 ($A[0..2]$)    | 8             |
| 4                    | 4 ($A[0..3]$)    | 9             |
| 5                    | 5 ($A[0..4]$)    | 14            |

带 0 的前缀和数组 $S = [0, 3, 4, 8, 9, 14]$。

**计算子数组 $A[2 \dots 4]$ 的和：**

$A[2 \dots 4] = [4, 1, 5]$，和为 $4 + 1 + 5 = 10$。

使用前缀和计算：



$$\text{Sum}(A[2 \dots 4]) = S[4+1] - S[2] = S[5] - S[2]$$

$$\text{Sum} = 14 - 4 = 10$$



结果正确！



# 差分算法

## 📚 差分算法介绍





### 什么是差分数组？



差分数组 $D$ 是针对原始数组 $A$ 而构建的一个数组。它的核心定义是：

- $D[i]$ 存储的是原始数组 $A$ 中**相邻两个元素的差值**。

用公式表示：

$$D[i] = A[i] - A[i-1] \quad (\text{其中 } A[-1] \text{ 视作 } 0)$$

换句话说，差分数组 $D$ 是原始数组 $A$ 的**一阶前向差分**。



### 核心思想：更新转化为点操作



差分算法的核心在于：

1. 构造关系： 原始数组 $A$ 是差分数组 $D$ 的前缀和数组。

   

   $$\text{由 } A[i] = D[i] + A[i-1] \text{ 可知，} A[i] = D[0] + D[1] + \dots + D[i]$$

2. **区间更新：** 如果要对原始数组 $A$ 的一个子区间 $A[i \dots j]$ **同时加上一个值 $C$**，我们只需要对差分数组 $D$ 中的**两个点**进行操作。

3. **最终求值：** 在所有区间更新操作完成后，通过对差分数组 $D$ 求前缀和，即可还原出最终更新后的数组 $A$。

这种技术将 $O(n)$ 的区间更新操作，优化成了 $O(1)$ 的差分数组点操作。

------



## 🔍 差分数组的构建与操作





### 1. 差分数组的构建（预处理）



给定一个原始数组 $A$，我们首先构建它的差分数组 $D$。



#### 💡 具体例子



假设原始数组 $A = [3, 5, 8, 2, 7]$ (长度 $n=5$)：

| **索引 i** | **A[i]** | **A[i−1] (或 0)** | **D[i]=A[i]−A[i−1]** |
| ---------- | -------- | ----------------- | -------------------- |
| 0          | 3        | $A[-1]=0$         | $3 - 0 = 3$          |
| 1          | 5        | $A[0]=3$          | $5 - 3 = 2$          |
| 2          | 8        | $A[1]=5$          | $8 - 5 = 3$          |
| 3          | 2        | $A[2]=8$          | $2 - 8 = -6$         |
| 4          | 7        | $A[3]=2$          | $7 - 2 = 5$          |

**差分数组 $D$:** $[3, 2, 3, -6, 5]$

**验证（求 $D$ 的前缀和）：**

- $D[0]=3 \implies A[0]=3$

- $D[0]+D[1]=3+2=5 \implies A[1]=5$

- $D[0]+D[1]+D[2]=5+3=8 \implies A[2]=8$

  ... 验证成功。



### 2. 区间更新操作（$O(1)$ 操作）



目标：对原始数组 $A$ 的子区间 $A[i \dots j]$ 的所有元素，都加上一个常数 $C$。

我们只需要对差分数组 $D$ 进行**两个修改**：

1. **在起点 $i$ 处加上 $C$：** $D[i] = D[i] + C$
   - 效果：由于 $A[i]$ 是 $D[0 \dots i]$ 的和，此操作会使 $A[i], A[i+1], \dots, A[n-1]$ **全部**增加 $C$。
2. **在终点 $j+1$ 处减去 $C$：** $D[j+1] = D[j+1] - C$
   - 效果：此操作会**抵消**从 $A[j+1]$ 开始的元素受到的影响。
   - $A[j+1] = D[0 \dots j]$ 的和 + $(D[j+1]-C)$ 的和。抵消了 $C$ 的增长。
   - $A[j+2], \dots$ 也会被抵消。

> **注意：** 如果 $j$ 是数组的最后一个索引 $(n-1)$，那么 $j+1=n$ 已经超出了 $D$ 的范围。此时，我们只需要进行第 1 步操作，因为不需要抵消后续元素。



#### 💡 具体例子（区间更新）



原始数组 $A = [3, 5, 8, 2, 7]$，差分数组 $D = [3, 2, 3, -6, 5]$。

**操作：** 对 $A[2 \dots 4]$（即 $[8, 2, 7]$）的每个元素都加上 $C=10$。

- 起始索引 $i=2$，结束索引 $j=4$。

1. 起点操作： $D[i] = D[2]$ 加上 $10$。

   

   $$D[2] = 3 + 10 = 13$$

2. **终点操作：** $j=4$ 是数组末尾， $j+1=5$ 超出范围，**无需进行** $D[j+1]$ 的操作。

**更新后的差分数组 $D'$:** $[3, 2, **13**, -6, 5]$



### 3. 还原最终数组（前缀和）



所有区间更新操作完成后，我们对 $D'$ 求前缀和，得到最终更新后的数组 $A'$。

| **索引 i** | **D′[i]** | **A′[i]=∑k=0iD′[k]** |
| ---------- | --------- | -------------------- |
| 0          | 3         | 3                    |
| 1          | 2         | $3 + 2 = 5$          |
| 2          | 13        | $5 + 13 = 18$        |
| 3          | -6        | $18 + (-6) = 12$     |
| 4          | 5         | $12 + 5 = 17$        |

**最终数组 $A'$:** $[3, 5, 18, 12, 17]$

**与期望结果对比：** 原始 $A=[3, 5, 8, 2, 7]$。对 $A[2 \dots 4]$ 加上 10：

- $A[0]$: $3$ (不变)

- $A[1]$: $5$ (不变)

- $A[2]$: $8 + 10 = 18$

- $A[3]$: $2 + 10 = 12$

- $A[4]$: $7 + 10 = 17$

  结果完全吻合！

## 🖥️ C++ 代码实现



C++

```cpp
#include <iostream>
#include <vector>
#include <numeric>

using namespace std;

/**
 * @brief 差分数组类，用于高效处理区间更新
 */
class DifferenceArray {
private:
    vector<int> diff; // 差分数组 D
    int n;            // 原始数组长度

public:
    // 构造函数：初始化差分数组
    DifferenceArray(const vector<int>& A) {
        n = A.size();
        diff.resize(n);

        if (n > 0) {
            // D[0] = A[0] - 0
            diff[0] = A[0]; 
            // D[i] = A[i] - A[i-1]
            for (int i = 1; i < n; ++i) {
                diff[i] = A[i] - A[i - 1];
            }
        }
    }

    /**
     * @brief 对原始数组 A[i..j] 区间加上一个值 C
     * @param i 起始索引 (0-based)
     * @param j 结束索引 (0-based)
     * @param C 要加上的值
     */
    void increment(int i, int j, int C) {
        // 1. 在起点 i 处增加 C
        diff[i] += C;

        // 2. 在终点 j+1 处减去 C (如果 j 不是数组末尾)
        // 注意：j+1 可能是数组末尾的索引 n
        if (j + 1 < n) {
            diff[j + 1] -= C;
        }
    }

    /**
     * @brief 还原更新后的原始数组 A'
     * @return vector<int> 最终数组 A'
     */
    vector<int> restoreArray() {
        vector<int> A_prime(n);
        if (n > 0) {
            // A'[0] = D[0]
            A_prime[0] = diff[0]; 
            // 从 A'[1] 开始，利用前缀和还原: A'[i] = A'[i-1] + D[i]
            for (int i = 1; i < n; ++i) {
                A_prime[i] = A_prime[i - 1] + diff[i];
            }
        }
        return A_prime;
    }
};

int main() {
    vector<int> A = {10, 5, 20, 15}; // 原始数组

    cout << "原始数组 A: [10, 5, 20, 15]" << endl;

    // 1. 初始化差分数组 (预处理)
    DifferenceArray da(A);

    // 2. 进行多次区间更新 (O(1) 操作)
    
    // 操作 1: A[1..2] 加上 5 (即 A[1] 和 A[2] 增加 5)
    cout << "\n操作 1: A[1..2] 加上 5" << endl;
    da.increment(1, 2, 5); // i=1, j=2, C=5
    // 差分数组 D 变化：D[1]+=5, D[3]-=5

    // 操作 2: A[0..0] 减去 3 (即 A[0] 减去 3)
    cout << "操作 2: A[0..0] 减去 3" << endl;
    da.increment(0, 0, -3); // i=0, j=0, C=-3
    // 差分数组 D 变化：D[0]-=3, D[1]+=3

    // 3. 还原最终数组 (求 D 的前缀和)
    vector<int> A_final = da.restoreArray();

    cout << "\n最终数组 A': [";
    for (int x : A_final) {
        cout << x << " ";
    }
    cout << "]" << endl;
    
    // 预期结果：
    // A[0]: 10 - 3 = 7
    // A[1]: 5 + 5 = 10
    // A[2]: 20 + 5 = 25
    // A[3]: 15 (不变)
    // 最终数组 A': [7, 10, 25, 15]

    return 0;
}
```

## 🌟 总结与应用





### 差分算法的特点



| **特点**     | **描述**                                              | **复杂度**                          |
| ------------ | ----------------------------------------------------- | ----------------------------------- |
| **构建**     | 预先计算差分数组 $D$。                                | $O(N)$                              |
| **区间更新** | 对 $A[i \dots j]$ 更新，仅需修改 $D[i]$ 和 $D[j+1]$。 | $O(1)$                              |
| **单点查询** | 必须通过求 $D$ 的前缀和来还原 $A[i]$。                | $O(N)$ (若多次查询，需 $O(N)$ 还原) |
| **应用场景** | 大量区间更新，少量或最后一次性查询。                  |                                     |



### 差分算法的应用场景



1. **大规模区间更新问题：** 这是差分数组最主要的应用场景。例如，有一系列活动，每个活动都会在一段时间内（对应数组的一个区间）对某个资源量（数组元素）产生影响，需要计算所有活动结束后资源的最终状态。
2. **二维差分：** 类似于二维前缀和，二维差分数组可以实现对矩阵中**矩形区域**的批量更新（加减一个常数），操作复杂度也降为 $O(1)$。
3. **差分约束：** 虽然名字相似，但差分约束系统是图论中的一种算法，用于求解形如 $X_i - X_j \le C_k$ 的不等式组，与这里的数组差分算法概念上不同，但都利用了差值关系。



# sort()

## 🌟 复杂对象排序的例子



假设我们有一个存储学生信息的结构体 `Student`：

C++

```
struct Student {
    string name;
    int score;
};
```

我们要实现一个排序规则：

1. **主要规则：** 按照 `score` **降序**排列（分数高的在前）。
2. **次要规则（分数相同时）：** 按照 `name` **升序**排列（姓名首字母小的在前）。

C++

```cpp
void complex_example() {
    vector<Student> students = {
        {"Alice", 90},
        {"Bob", 85},
        {"Charlie", 90},
        {"David", 95}
    };
    
    sort(students.begin(), students.end(), [](const Student& s1, const Student& s2) -> bool {
        // 1. 主要规则：分数降序 (s1.score > s2.score)
        if (s1.score != s2.score) {
            // 如果分数不相等，返回分数高的排在前面
            return s1.score > s2.score;
        } else {
            // 2. 次要规则：分数相等时，姓名升序 (s1.name < s2.name)
            // 姓名是 string 类型，可以直接用 < 进行字典序比较
            return s1.name < s2.name;
        }
    });

    // 预期输出：
    // David (95)
    // Alice (90)  <-- 'A' 比 'C' 小，排在 Charlie 前面
    // Charlie (90)
    // Bob (85)
    cout << "复杂对象排序结果 (分数降序，姓名升序):" << endl;
    for (const auto& s : students) {
        cout << s.name << " (" << s.score << ")" << endl;
    }
}
```

# 离散化

## 🧐 什么是离散化算法？





### 核心思想



当数据量 $N$ 不大（例如 $10^5$ 级别），但数据的值域 $V$ 非常大（例如 $10^9$ 级别）时，如果我们需要对数据的**值**进行统计、排序、或作为数组/数据结构的**索引**，直接使用原值作为索引会导致内存浪费或计算效率低下。

**离散化**就是将这些巨大的原始数值，替换成它们在所有值中的**相对大小排名**。

- **目的：** 将原本稀疏、庞大的值域 $[1, 10^9]$ 压缩为连续、紧凑的值域 $[1, N]$（其中 $N \le 10^5$）。



### 为什么需要离散化？



| **特性**      | **原始数据值域（[1,109]）**                      | **离散化后的值域（[1,N]）**      |
| ------------- | ------------------------------------------------ | -------------------------------- |
| **大小**      | 巨大、稀疏                                       | 紧凑、小巧                       |
| **内存/效率** | 难以作为数组下标，可能需要哈希表或浪费大量内存。 | 可以直接作为数组下标，操作高效。 |



### 适用场景



离散化常用于需要利用**数据的值**作为**下标**或进行**排名比较**的算法中，例如：

1. **统计/计数：** 您提出的问题。
2. **树状数组/线段树：** 当需要在巨大值域上进行区间操作时。
3. **坐标压缩：** 在几何问题中，将巨大坐标压缩到小的离散坐标上。

------



## 🎯 解决您的具体问题：统计出现次数



**问题：** 输入不大于 $10^5$ 个不大于 $10^9$ 的数，统计每个数的出现次数。

在这个问题中：

- 数据量 $N \le 10^5$。
- 值域 $V \le 10^9$。



### 方案对比



| **方案**                               | **优点**                       | **缺点**                                                 | **适用于本题？**         |
| -------------------------------------- | ------------------------------ | -------------------------------------------------------- | ------------------------ |
| **1. 离散化 + 数组**                   | 统计和查询效率极高 ($O(1)$)    | 需要额外的排序和映射步骤                                 | **是，但不如哈希表简单** |
| **2. 哈希表 (std::map/unordered_map)** | 直接使用原值作为键，代码最简洁 | 理论上 $O(N \log N)$ (map) 或平均 $O(N)$ (unordered_map) | **最简单高效的方案**     |
| **3. 直接用数组**                      | 效率 $O(1)$                    | 内存消耗过大，无法创建 $10^9$ 大小的数组                 | **不可行**               |

虽然哈希表是最简单直接的方案，但为了讲解离散化，我们先展示如何使用离散化来解决这个问题，然后再对比哈希表。



### 🌟 离散化算法步骤（用于计数）



使用离散化解决此问题的步骤是：



#### 步骤 1: 存储并去重 (O(N log N))



将所有输入的 $N$ 个数存入一个数组 `A` 中，然后复制到另一个数组 `B` 中，并对 `B` 进行**排序**和**去重**。

C++

```
// 假设 input_data 是原始输入
vector<int> A = {1000000000, 1, 5, 1, 1000000000, 5};
vector<int> B = A; // 复制所有元素

// 排序
sort(B.begin(), B.end()); // B: {1, 1, 5, 5, 10^9, 10^9}

// 去重 (unique 返回去重后有效元素的尾后迭代器)
B.erase(unique(B.begin(), B.end()), B.end()); 
// B: {1, 5, 1000000000} (去重后，N' <= 10^5)
```



#### 步骤 2: 建立映射关系 (O(1))



现在，数组 `B` 包含了所有不同的原始值，且它们是升序排列的。`B` 的下标就是我们所需的**离散化值**（排名）。

- 原值 $V$ 对应的离散化值 $R$ 满足 $B[R-1] = V$。



#### 步骤 3: 统计计数 (O(N log N))



创建一个大小为 $N'$ 的计数数组 `count_arr`（$N'$ 是 `B` 的长度）。遍历原始数组 `A`，将每个元素 $A[i]$ 映射到它的离散化值，并在 `count_arr` 中对应位置计数。

**如何快速找到映射关系？** 使用 `std::lower_bound` (二分查找)。

C++

```
// count_arr 负责存储每个离散化值（排名）对应的计数
vector<int> count_arr(B.size() + 1, 0); 

for (int val : A) {
    // 找到 val 在有序去重数组 B 中的位置 (迭代器)
    auto it = lower_bound(B.begin(), B.end(), val);
    
    // 计算它对应的索引 (0-based index)
    int rank = distance(B.begin(), it); 
    
    // rank 就是离散化后的值 (0 ~ N'-1)
    
    // 计数
    count_arr[rank]++; 
}

// 最终结果：
// count_arr[0] 存储的是原值 B[0] (即 1) 的出现次数
// count_arr[1] 存储的是原值 B[1] (即 5) 的出现次数
// count_arr[2] 存储的是原值 B[2] (即 10^9) 的出现次数
```



### 完整 C++ 代码（离散化+数组）



C++

```cpp
#include <iostream>
#include <vector>
#include <algorithm>
#include <iterator>

using namespace std;

// 离散化主函数
void solve_discretization(vector<int>& A) {
    if (A.empty()) return;

    // 1. 复制、排序、去重 -> 得到离散值集合 B
    vector<int> B = A;
    sort(B.begin(), B.end());
    B.erase(unique(B.begin(), B.end()), B.end());

    int N_prime = B.size(); // 离散值数量

    // 2. 创建计数数组，大小为去重后的元素个数
    vector<int> count_arr(N_prime, 0);

    // 3. 遍历原始数组 A，进行映射和统计
    cout << "--- 统计结果 (使用离散化数组) ---" << endl;
    for (int val : A) {
        // 使用二分查找找到 val 在 B 中的位置
        auto it = lower_bound(B.begin(), B.end(), val);
        
        // rank 是 val 在 B 中的 0-based index，即离散化后的值
        int rank = distance(B.begin(), it); 
        
        // 统计计数
        count_arr[rank]++; 
    }

    // 4. 输出结果
    for (int i = 0; i < N_prime; ++i) {
        cout << "原值 " << B[i] << " 出现次数: " << count_arr[i] << endl;
    }
}
```

# 哈希表

## 🎯 一、 哈希表的基本原理



哈希表的思想非常简单：**通过一个函数，将键（Key）直接映射到数组的下标（索引）上。**



### 1. 组成部分



哈希表主要由三部分构成：

| **部分**                               | **作用**                                                  | **对应到数组** |
| -------------------------------------- | --------------------------------------------------------- | -------------- |
| **键 (Key)**                           | 待存储和查找的唯一标识（可以是数字、字符串、对象等）。    | 输入           |
| **哈希函数 (Hash Function)**           | 将任意大小的键值转换为固定范围内的**整数**。              | 运算过程       |
| **哈希数组 (Hash Array/Bucket Array)** | 实际存储值（Value）的数组，哈希函数的输出就是数组的下标。 | 输出结果       |



### 2. 工作流程



1. **存储/插入：**
   - 将待存储的键 `key` 通过**哈希函数** $H$ 计算得到一个哈希值 $h = H(\text{key})$。
   - 将哈希值 $h$ 通过取模运算 $\text{index} = h \pmod M$（$M$ 是哈希数组的长度）映射到数组的合法下标。
   - 将 `(key, value)` 对存入该下标对应的位置。
2. **查找：**
   - 输入待查找的键 `key`。
   - 使用**同样的哈希函数**计算出对应的数组下标 $\text{index}$。
   - 直接访问数组 $\text{HashArray}[\text{index}]$ 即可快速取出对应的值 `value`。

------



## 🔑 二、 核心组件：哈希函数 (Hash Function)



哈希函数的质量直接决定了哈希表的性能。一个好的哈希函数需要满足以下特性：

1. **快速计算：** 能够在 $O(1)$ 或极快的时间内计算出哈希值。
2. **均匀分布 (Uniformity)：** 能够将不同的键值均匀地分散到哈希数组的各个位置，减少“堆积”现象。
3. **确定性：** 相同的输入键必须总是产生相同的哈希值。

在实际实现中，通常会结合多种方法，如乘法、位操作、除法（取模）等，来构建一个高效的哈希函数。

------



## 💥 三、 哈希冲突 (Hash Collision)





### 什么是冲突？



由于键值空间 $V$（例如 $10^{18}$ 种字符串）远远大于哈希数组空间 $M$（例如 $10^5$ 个桶），根据**鸽巢原理**，必然存在不同的键 $K_1 \ne K_2$，但它们经过哈希函数计算后，却映射到了**同一个数组下标**：

$$H(K_1) \pmod M = H(K_2) \pmod M$$

这种情况就是**哈希冲突**。哈希冲突是不可避免的，因此处理冲突的方法是哈希表设计的关键。



### 冲突的常见解决方法



处理冲突主要有两种主流方法：



#### 1. 链式地址法 (Chaining)



这是最常用、实现最简单的方案（C++ 的 `std::unordered_map` 和 Java 的 `HashMap` 多数采用此方法）。

- **原理：** 将哈希数组的每一个位置（桶）视为一个**链表的头节点**。
- **实现：** 当发生冲突时，将新的键值对作为链表的新节点，挂载到该哈希桶对应的链表末尾。
- **操作：**
  - **插入：** 计算索引，将新元素插入链表的头部或尾部 ($O(1)$)。
  - **查找：** 计算索引，然后**遍历**该链表，直到找到匹配的键 ($O(1)$ 平均；最坏 $O(N)$)。
- **特点：** 允许哈希数组的负载因子（元素总数 / 桶数）大于 1。



#### 2. 开放寻址法 (Open Addressing)



- **原理：** 当哈希函数计算出的位置已被占用时，**探查**（Probe）哈希数组中的其他空闲位置来存放新元素。
- **实现：** 不需要额外的链表结构，所有元素都直接存放在哈希数组中。
- **探查策略：**
  - **线性探查 (Linear Probing)：** 向后以固定步长（通常是 1）依次寻找下一个空桶。
  - **二次探查 (Quadratic Probing)：** 向后以平方步长（$1^2, 2^2, 3^2, \dots$）寻找空桶。
- **操作：** 无论是插入、查找还是删除，都需要沿着探查序列进行。
- **特点：** 数组不能被完全填满，通常要求负载因子小于 0.5 或 0.75。删除操作复杂（需要“懒删除”或标记为 $\text{TOMBSTONE}$）。

------



## 📈 四、 哈希表的性能和优化





### 1. 负载因子 (Load Factor)



负载因子 $\alpha$ 是衡量哈希表满载程度的关键指标：



$$\alpha = \frac{\text{元素数量 (N)}}{\text{桶的数量 (M)}}$$

- **链式地址法：** 当 $\alpha$ 增大时，链表变长，查找效率下降。
- **开放寻址法：** 当 $\alpha$ 接近 1 时，连续空闲桶越来越少，聚集现象严重，性能急剧下降。



### 2. 扩容 (Resizing/Rehashing)



当负载因子 $\alpha$ 超过一个阈值（例如 0.75）时，为了维持 $O(1)$ 的性能，需要进行**扩容**：

1. 创建一个**两倍**于原大小的新哈希数组。
2. 对原数组中的所有元素**重新计算哈希值**（Rehashing），并将它们插入到新数组中。
3. 这个过程的时间复杂度是 $O(N)$，但由于它不经常发生，所以**平均**时间复杂度仍然是 $O(1)$。

------



## 总结：为什么哈希表如此重要？



哈希表是实现**字典 (Dictionary)** 或**映射 (Map)** 接口的最佳方式，它在许多领域都有应用：

| **领域**     | **应用实例**                                                 |
| ------------ | ------------------------------------------------------------ |
| **编程语言** | C++ 的 `std::unordered_map`，Python 的 `dict`，Java 的 `HashMap` 都是基于哈希表实现的。 |
| **数据库**   | 用于索引数据记录，实现快速查找。                             |
| **缓存**     | Web 服务器或应用程序中的缓存机制（如 Redis）大量使用哈希表。 |
| **算法竞赛** | 统计元素出现次数、查找重复元素、解决两数之和等问题时，哈希表是首选工具。 |
