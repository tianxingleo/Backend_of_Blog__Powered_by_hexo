> Lecture 01 Overview of Computer Graphics：
>
> **课程主讲人**：闫令琪，清华大学本科，System Professor，研究方向是图形学中的渲染和绘制。
>
> **课程助教**：刘广哲、石宇辰、邓俊辰，后续会根据课程需求添加更多助教。
>
> **课程内容**：
>
> 1.  **什么是图形学？为什么要学？**
>     *   图形学的定义：合成和操作视觉信息。
>     *   应用领域广泛：
>         *   **游戏**：如《只狼》（全局光照，画面亮度是好画面的标准）、《无主之地三》（卡通风格）。
>         *   **电影**：如《黑客帝国》（特效，如子弹时间）、《阿凡达》（面部和动作捕捉）。
>         *   **动画**：如《疯狂动物城》（毛发渲染）、《冰雪奇缘》（烟雾、粒子、衣服、头发模拟）。
>         *   **设计**：如CAD（Computer-Aided Design），汽车设计、室内设计（宜家网站75%的图是渲染的）。
>         *   **可视化**：将三维扫描、CT、核磁共振等数据转化为视觉信息，如地理数据、医学成像。
>         *   **虚拟现实（VR）与增强现实（AR）**：VR是完全虚拟的，AR是现实与虚拟结合，如微软HoloLens设计摩托车。
>         *   **数字绘画与图像编辑**：如iPad手绘、Adobe Photoshop（曲线、差值、透明度等）。
>         *   **模拟/仿真**：如沙尘暴、房屋破坏、黑洞（光线偏折）。
>         *   **图形用户接口（GUI）**：操作系统界面设计（Mac、Windows、Linux）。
>         *   **字体（Typography）**：字体在放大后仍清晰的原理（点阵与矢量图的区别）。
>     *   图形学很难，因为它需要对真实世界的理解（物质、光、材质）和新的计算、显示方法。
>
> 2.  **课程具体包括哪些内容？**
>     *   **光栅化（Rasterization）**：将三维几何形体显示在屏幕上，是实时图形学（每秒30帧，30 FPS）的主要应用，主要用于游戏。
>     *   **几何（Geometry）**：如何表示光滑曲线、曲面，通过细分得到复杂曲面，形状变化时如何保持拓扑结构。
>     *   **光线追踪（Ray Tracing）**：动画和电影中生成真实感画面的方法，效果好但速度慢，目前有实时光线追踪技术。
>     *   **动画与模拟（Animation & Simulation）**：模拟弹性球落地、布料在风中摆动、沙尘暴、黑洞等物理过程。
>
> 3.  **课程不包括哪些内容？**
>     *   **硬件编程**：不讲Open GL、DirectX、Vulkan等图形API的具体使用，而是讲解背后的原理，让学生能自行学习API。
>     *   **三维建模或游戏开发**：不教Maya、Max等建模工具，也不教Unity、Unreal等游戏引擎的使用，因为这些是工具层面的应用。
>     *   **计算机视觉**：不涉及图像识别、语义分割、三维重建等需要“猜测”和“理解”图像内容的部分，也不特别提及深度学习在图像生成和分析中的应用，因为这些属于图像层面而非三维模型层面的操作。
>
> **图形学、计算机视觉、图像/视频处理、计算摄影学之间的关系**：
> *   **图形学（Graphics）**：从三维模型（Model）生成图像（Image），或在模型层面进行模拟和操作。
> *   **计算机视觉（Computer Vision）**：从图像（Image）识别、分析、理解三维模型（Model），涉及猜测和推断。
> *   **图像/视频处理（Image/Video Processing）**：在图像（Image）层面进行操作和转换。
> *   **计算摄影学（Computational Photography）**：通过不同的摄像和捕捉方法，测量光学属性，拍摄特殊图像。
> *   这些领域边界越来越模糊，相互结合紧密。
>

> Lecture 02 Review of Linear Algebra：
>
> 1.  **课程资料与访问方式**：每节课后，课件将以PDF形式发布，录播视频也会上传至GAMES网站。两者均可通过课程主页访问。
> 2.  **推荐阅读材料**：课程内容与“虎书”（The Tiger Book）中的某些章节相似，建议学生在课前或课后阅读相关章节，但这并非强制要求。
> 3.  **课程结构回顾**：课程分为四部分：光栅化、几何、光线追踪和模拟/动画。
> 4.  **图形学基础**：本节课主要介绍图形学的基础，特别是线性代数。
> 5.  **图形学依赖的知识领域**：图形学依赖于基础数学（线性代数、微积分、统计）、物理（光学、力学，包括波动光学等）、信号处理、数值分析以及美学。
> 6.  **向量概念**：向量表示方向和长度，平移不改变向量。单位向量长度为1，常用于表示方向。
> 7.  **向量基本操作**：
>     *   **向量求和**：可通过平行四边形法则或三角形法则进行几何理解，代数上直接将坐标相加。
>     *   **坐标表示**：在直角坐标系（笛卡尔坐标系）中，向量可用其在各轴上的分量表示，默认是列向量。
>     *   **向量长度**：在直角坐标系中，向量长度通过勾股定理计算。
> 8.  **向量点乘（Dot Product）**：
>     *   **定义**：a · b = |a| |b| cos(θ)，结果是一个标量（数字）。
>     *   **用途**：
>         *   计算两个向量之间的夹角余弦，进而得到夹角。
>         *   特别适用于单位向量，此时点乘结果直接是夹角余弦。
>         *   计算一个向量在另一个向量上的投影。
>         *   判定两个向量方向的接近程度（点乘结果接近1表示方向一致，接近-1表示方向相反，0表示垂直）。
>         *   判定方向的前后关系（正数表示大致向前，负数表示大致向后，零表示垂直）。
>     *   **代数计算**：对应元素相乘再相加（例如，二维向量 (xa, ya) · (xb, yb) = xa*xb + ya*yb）。
>     *   **性质**：满足交换律、结合律和分配律。
> 9.  **向量叉乘（Cross Product）**：
>     *   **定义**：a × b 得到一个新的向量，其方向垂直于a和b所在的平面，大小为 |a| |b| sin(θ)。
>     *   **方向判定**：使用右手螺旋定则（从a向b旋转，拇指方向即为叉乘结果的方向）。
>     *   **性质**：
>         *   不满足交换律：a × b = - (b × a)。
>         *   向量叉乘自身结果为零向量。
>         *   满足分配律和结合律。
>     *   **用途**：
>         *   建立三维直角坐标系（例如，x轴 × y轴 = z轴）。
>         *   判定左右关系（例如，在二维平面上，a × b的z分量为正表示b在a的左侧）。
>         *   判定内与外关系（例如，通过判断一个点是否始终在三角形所有边的同一侧来确定它是否在三角形内部）。
> 10. **矩阵概念**：
>     *   **定义**：按行和列排列的一组数。
>     *   **矩阵乘法**：
>         *   要求第一个矩阵的列数等于第二个矩阵的行数。
>         *   结果矩阵的每个元素是通过第一个矩阵对应行与第二个矩阵对应列的点乘计算得出。
>         *   不满足交换律（A * B ≠ B * A），但满足结合律和分配律。
> 11. **矩阵操作**：
>     *   **转置**：将矩阵的行和列互换（(AB)T = BT AT）。
>     *   **单位矩阵（Identity Matrix）**：对角线上元素为1，其余为0的方阵，与任何矩阵相乘结果不变（I * A = A）。
>     *   **逆矩阵（Inverse Matrix）**：如果A * A⁻¹ = I，则A⁻¹是A的逆矩阵（(AB)⁻¹ = B⁻¹ A⁻¹）。
> 12. **向量与矩阵的结合**：
>     *   向量点乘可以表示为矩阵乘法：a · b = aᵀb。
>     *   向量叉乘可以表示为矩阵乘法：a × b = A* b（其中A* 是a向量的对偶矩阵）。
